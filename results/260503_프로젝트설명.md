# DCC (Data Collection Coin) 프로젝트 상세 설명 보고서

이 보고서는 현재 진행 중인 'DCC (Data Collection Coin)' 블록체인 프로젝트의 구조, 주요 기능 및 기술적 세부 사항을 상세히 설명합니다.

## 1. 프로젝트 개요
DCC 프로젝트는 데이터 수집 및 거래를 위한 탈중앙화 생태계를 구축하는 것을 목표로 합니다. 사용자는 데이터를 제공하고 보상으로 DCC 토큰을 받으며, 데이터 구매자는 DCC 토큰을 지불하여 데이터를 획득합니다. 이 과정에서 토큰의 가치 유지를 위해 소각(Burn) 메커니즘이 통합되어 있습니다.

- **프로젝트 명칭**: DCC (Data Collection Coin)
- **네트워크**: Polygon (Mainnet / Amoy Testnet)
- **토큰 표준**: ERC20

## 2. 주요 스마트 계약 분석

### 2.1 DCC.sol (기본 토큰 계약)
DCC 생태계의 핵심이 되는 ERC20 토큰 계약입니다.
- **주요 기능**:
  - `mint(address to, uint256 amount)`: 관리자 권한을 가진 주소가 새로운 토큰을 발행합니다.
  - `burn(address from, uint256 amount)`: 관리자 권한을 가진 주소가 특정 주소의 토큰을 소각합니다.
- **권한 관리 (AccessControl)**:
  - `DEFAULT_ADMIN_ROLE`: 계약의 전반적인 관리 권한.
  - `MINTER_ROLE`: 토큰 발행 권한.
  - `BURNER_ROLE`: 토큰 소각 권한.

### 2.2 BurnAndMintManager.sol (서비스 매니저 계약)
비즈니스 로직을 담당하며, 토큰의 유통량을 조절하고 보상을 관리합니다.
- **주요 기능**:
  - `purchaseData(uint256 amount)`: 사용자가 데이터를 구매할 때 호출합니다. 설정된 `burnRate`(기본 50%)에 따라 결제 금액의 일부를 소각하고 나머지는 계약에 보관합니다.
  - `rewardProvider(address to, uint256 amount)`: 데이터 제공자에게 보상으로 새로운 토큰을 발행합니다.
  - `setBurnRate(uint256 _rate)`: 관리자가 데이터 구매 시의 소각 비율을 조정합니다.

## 3. 주요 프로세스

### 3.1 데이터 구매 및 소각 Flow
1. 구매자가 `DCC` 토큰을 `BurnAndMintManager` 계약에 사용할 수 있도록 `approve` 합니다.
2. 구매자가 `purchaseData` 함수를 실행합니다.
3. 계약은 토큰을 가져온 뒤, 지정된 비율(예: 50%)만큼 즉시 소각하여 전체 공급량을 줄입니다.
4. 소각되지 않은 나머지 토큰은 시스템 운영이나 향후 활용을 위해 계약 내에 보관됩니다.

### 3.2 데이터 제공자 보상 Flow
1. 관리자가 검증된 데이터 제공자에게 `rewardProvider` 함수를 통해 DCC 토큰을 새로 발행하여 지급합니다.
2. 이를 통해 유동성이 공급되며 데이터 수집 활동을 장려합니다.

## 5. 프로젝트 특징 및 장점
- **리브랜딩 완료**: 이전 DCT(Data Collection Token)에서 DCC로 성공적으로 전환되었습니다.
